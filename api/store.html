Store¶

class ZCloudStore¶

Stores represent a service that provides persistence of arbitrary blobs. An example is Amazon S3.

setup(suffix, n_parameters, parameters, error)¶
Protected method - used during object construction.
create(address, progress, error)¶

Creates whatever is represented by the address.

Parameters:
  • address (String) – the address to create
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
upload(address, producer, progress, error)¶

Upload some data. producer will be consulted for metadata (e.g. MD5 hash and size) and is responsible for actually reading the data to be uploaded. The Store handles actually uploading the data to the storage service.

Parameters:
  • address (String) – the address to store the data at
  • producer – an UploadProducer
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
download(address, consumer, progress, error)¶

Download data from address. The consumer is responsible for “doing something useful” with the data received from the storage service. For example, it could save it to a file.

Parameters:
  • address (String) – the address to fetch data from
  • consumer – a DownloadConsumer
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean

list(template, consumer, progress, error)¶

List all addresses matching template, calling List all addresses matching template, calling ListConsumer.got_result() for each one. The template should just use %s as a placeholder and %% to represent %.

Parameters:
  • template – the template to list matches for
  • consumer – a DownloadConsumer
  • progress – an optional ProgressListener
  • error – optional storage for an error

true (on success) or false (on failure)
